Implementation
- Added LittleAgentsExtension/Llm/OpenAiChatClient.cs implementing ILlmChatClient.
- Constructs OpenAI.Chat.ChatClient with model, ApiKeyCredential(apiKey), and OpenAIClientOptions { Endpoint = new Uri(baseUrl) }, where baseUrl is provider.BaseUrl passed verbatim into the helper.
- Does not append /v1 and does not manually concatenate URLs.
- Converts LittleAgentsExtension.Llm.ChatMessage[] to Microsoft.Extensions.AI.ChatMessage[] with System/User/Assistant role mapping.
- Streams client.GetStreamingResponseAsync(messages, new ChatOptions { Temperature = (float?)request.Temperature }, ct) and yields only non-empty update.Text chunks.
- Wraps construction failures as InvalidOperationException("Provider misconfigured: ...") after scrubbing exception text with (?i)(bearer\s+)?sk-[A-Za-z0-9_-]{4,} -> ***.
- Does not expose SDK types through ILlmChatClient.

SDK API notes / deviations
- Local package XML for Microsoft.Extensions.AI.OpenAI 10.7.0 confirms OpenAIClientExtensions.AsIChatClient(OpenAI.Chat.ChatClient) and OpenAIChatClient.GetStreamingResponseAsync(...).
- Context7 /openai/openai-dotnet docs confirm ChatClient(model, ApiKeyCredential, OpenAIClientOptions { Endpoint = new Uri(...) }) for custom endpoint.
- Deviation from plan: IAsyncEnumerable<ChatResponseUpdate> in this target did not provide ConfigureAwait(false), so the final code enumerates the SDK async stream directly while still passing ct into GetStreamingResponseAsync.
