Addendum 2026-06-28: real CmdPal Manage Providers no-op fix

Observed issue:
- During T44 manual setup, clicking `Manage Providers` in the real Command Palette UI produced no navigation.

Root cause:
- The pinned rows used `InvokableCommand` plus `CommandResult.GoToPage(new PinnedPageNavigationArgs(page) { PageId = ... })`.
- Upstream `GoToPageArgs` only carries `PageId` and `NavigationMode`; it does not carry an arbitrary target page object.
- The old unit test recursively found the hidden page object on the custom args type, so it passed even though CmdPal could not resolve the page in the real host.

Fix:
- `Manage Providers`, `+ New Agent`, and the DEBUG-only markdown ticker pinned rows now use the toolkit's direct-page pattern: `new ListItem(new SomePage())`.
- The zero-provider `+ New Agent` path now navigates directly to `ProviderEditFormPage` and shows `Add a provider first` as the row subtitle rather than relying on a nested toast result.
- Removed `PinnedNavigationCommand` and `PinnedPageNavigationArgs`.

Verification:

- Focused test after fix: `AgentsListPagePinnedNavigationTests` passed 4/4.

Addendum 2026-06-28: real CmdPal agent row no-op fix
