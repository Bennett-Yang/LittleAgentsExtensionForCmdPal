## Root cause (confirmed 2026-06-21T16:xx+08:00)
- Mechanism: The currently wired old placeholder page returned a `ListItem` whose command was `NoOpCommand`, leaving command identity empty/default. During CmdPal reload, the host initializes `ListItemViewModel` and reads `ICommand.Id`; this host path failed through the COM/WinRT proxy and tore down the extension object (`0x800706BE`, then repeated `0x800706BA`). Replacing the command with an `InvokableCommand` that has stable `Id` and `Name` lets the host read command metadata and the page renders.
- Evidence: User reproduced crash twice before the H2 toggle, then reported `Reload 是否还崩溃: no` and `TODO placeholder 页面: yes` after `TemplatePlaceholderCommand` with stable Id/Name.
- Toggle proof: With `NoOpCommand`, reload produced `ICommandMethods.get_Id` COMException. With explicit `TemplatePlaceholderCommand` stable Id/Name, reload did not crash. H1 toggle (`AnonymousCommand(null)`) did not change the failure.
- Fix scope: stable command classes in `LittleAgentsExtensionPage.cs` and `AgentsListPage.cs`.
